## Test structure

Tests follow a **modular structure**: each module contains its own tests inside a `tests/` folder next to the code they test.

```
core/
├── endpoints/tests/        # REST endpoint tests
├── ingest/tests/           # Ingestion pipeline tests
├── metrics/tests/          # Prometheus metrics tests
├── paths/tests/            # Path resolution tests
├── resilience/tests/       # Circuit breaker tests
├── server/tests/           # FastAPI server tests
└── tests/                  # General core tests

memory/
├── embeddings/tests/       # Embeddings and chunking tests
├── memory/tests/           # Memory system tests
├── rag/tests/              # RAG engine tests
└── shared/tests/           # Shared cache tests

personality/
├── events/tests/           # Event system tests
├── i18n/tests/             # Internationalization tests
├── loading/tests/          # Module loader tests
├── models/tests/           # Personality profile tests
└── module_manager/tests/   # Module manager tests

plugins/
├── llama_cpp_module/tests/ # Llama.cpp tests
├── mlx_module/tests/       # MLX tests
├── ollama_module/tests/    # Ollama tests
├── security/tests/         # Security tests
├── security_logger/tests/  # Security logger tests
└── web_ui_module/tests/    # Web UI tests

tests/                      # Global integration tests
```

## Running tests

```bash
# All unit tests
pytest

# A specific module
pytest plugins/ollama_module/tests/

# With coverage
pytest --cov=. --cov-report=html

# Full tests (including integration)
pytest -c pytest-full.ini
```

## Test types

### Unit tests
- Cover individual functions and classes
- Use mocks to isolate external dependencies (Qdrant, Ollama, MLX)
- Run quickly without external services

### Integration tests
- Verify interaction between modules
- Some require active services (Qdrant, Ollama)
- Located in `tests/integration/`

## Philosophy

- **Tests inside each module**: if you move a plugin to another repository, its tests go with it
- **Mocks by default**: unit tests don't depend on external services
- **Coverage > 90%**: minimum target for all modules
- **CI-ready**: all tests can run in a continuous integration environment
